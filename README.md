# Green Beacons Consulting Website

A modern, responsive static website built for **Green Beacons Consulting**, a company focused on infrastructure solutions for human development.

🌐 Live Website: https://greenbeacons.vercel.app/

---

## About the Project

This website serves as the digital presence of Green Beacons Consulting. It presents the company's mission, services, and contact information through a clean, responsive user interface.

The website is built using only front-end technologies:

- HTML5
- CSS3
- JavaScript (Vanilla JavaScript)

No frameworks, build tools, or external dependencies are required to run or modify the project.

---

## Features

- Responsive design across different screen sizes
- Modern and clean user interface
- Smooth scrolling and interactive elements
- Mobile-friendly navigation
- Optimized typography and layout
- Lightweight and fast-loading static website

## Getting Started

### 1. Clone the Repository

```bash
git clone <repository-url>
```

### 2. Navigate into the Project Folder

```bash
cd green-beacons
```

### 3. Run the Website Locally

Since this is a static website, there are no dependencies to install.

You can open the `index.html` file directly in your browser or use a local development server.

For Visual Studio Code users:

1. Install the **Live Server** extension.
2. Right-click on `index.html`.
3. Select **Open with Live Server**.

The website will open automatically in your default browser.

## Contributing

Contributions are welcome. Since this is a lightweight static website, the contribution process is simple.

### Steps to Contribute

1. Fork the repository.

2. Create a new branch:

```bash
git checkout -b feature/your-feature-name
```

3. Make your changes while maintaining the existing code style.

4. Test your changes:

- Ensure the website is responsive on desktop, tablet, and mobile devices.
- Verify all navigation links and buttons work correctly.
- Test the website on modern browsers.

5. Commit your changes:

```bash
git commit -m "Describe your changes"
```

6. Push your changes:

```bash
git push origin feature/your-feature-name
```

7. Open a Pull Request with a clear explanation of the changes made.
